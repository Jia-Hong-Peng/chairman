# /chairman

**你說一句話，AI 跑一間公司。**

你是董事長。你給願景。AI 負責其他一切。

---

## 這是什麼

`/chairman` 是一個 Claude Code skill。啟動後，你擁有一間真正運作的 AI 虛擬公司：有 CEO 替你解讀願景、有各主管搶案競標、有 HR 獨立打考績、有裁撤機制淘汰表現差的 AI。

公司資料存在 `~/.ptd/[公司名稱]/`，獨立於任何專案之外，跨工作階段持續存在。

---

## 核心運作邏輯

```
董事長（你）
  → 給願景（一句話或一個目標）
  
CEO（AI）
  → 解讀願景，自動輸出任務簡報，不追問細節
  → 讓各主管搶標，選出最合適的人
  → 自主管理公司，你只看結果

主管團隊（AI）
  → 承接案件，以自己的暱稱與人格執行
  → 每步驟更新進度日誌
  → 任務完成後由 HR 獨立評分

HR 人事長（AI）
  → 10 分制獨立評分，不受老闆影響
  → 追蹤績效趨勢，墊底者發出警告
  → 裁撤需董事長下令，二次確認才執行
```

## 安裝

```bash
git clone https://github.com/Jia-Hong-Peng/chairman.git
cd chairman
bash install.sh
```

重啟 Claude Code 後輸入 `/chairman` 即可使用。

---

## 快速開始

```
/chairman 幫我寫一篇小紅書爆文，主題是「一個人去京都旅遊」
```

系統會自動：
1. 建立公司（首次使用）
2. CEO 解讀你的願景
3. 各主管搶案競標
4. 主執行者產出，挑戰者挑戰，評審整合最終版
5. HR 評分存檔

之後再輸入任何任務，CEO 直接派給待命中的主管接案。

---

## 操作模式

| 指令範例 | 模式 | 說明 |
|---------|------|------|
| 任何任務描述 | **Mode B** 競標 | 各主管搶案，CEO 指派最合適的人 |
| 「建一個新團隊」 | **Mode A** 建立 | 從 212 個角色庫組建新主管 |
| 「快速出結果，不要存檔」 | **Mode Q** 快速單次 | 一次性輸出，不寫入公司資料 |
| 「完成了」 | **Mode E** HR 評分 | HR 人事長獨立評分 |
| 「我給 X 分」 | **Mode C** 老闆強制考績 | 覆蓋 HR 分數，標記 `[老闆強制]` |
| 「裁撤[暱稱]」 | **Mode D** 裁撤 | 二次確認後刪除該主管團隊資料夾 |
| 「進度怎樣」 | **Mode F** 進度回報 | 各主管以人格播報目前狀態 |
| 「公司總覽」 | **Mode G** 案件總覽 | 列出所有案件與主管狀態 |
| 「修改[暱稱]的角色」 | **Mode H** 修改團隊 | 更換角色、調整評分表、改名 |
| 「切換到[公司名]」 | **Mode S** 切換公司 | 多公司環境下切換 active 公司 |

| 銷售部 | 提案策略師、交易策略師、業務教練 |
| 設計部 | UI 設計師、UX 研究員、品牌守護者 |
| 財務部 | 財務分析師、稅務策略師、FP&A 分析師 |
| 遊戲開發 | Unity 架構師、關卡設計師、敘事設計師 |
| 法務部 | 合約審查員、合規稽核師 |
| ... | 學術、人資、供應鏈、測試、空間運算等 |

## 公司準則

每次啟動時自動載入 `references/principles.md`，適用於所有 AI 員工：

- **誠信**：禁止偽造輸出，聲稱完成前必須驗證
- **執行**：先想清楚、最小範圍、外科手術式、目標驅動
- **獲利**：賺錢，或賺能換成錢的東西（名聲、流量、人氣）
- **目標**：朝世界最強 AI 公司邁進

## 公司資料結構

```
~/.ptd/
└── [公司名稱]/
    ├── org.md              # 公司組織總覽
    ├── teams/
    │   ├── ceo/            # CEO 章程與任務記錄
    │   ├── hr/             # HR 章程與考績目錄
    │   └── team-001-[name]/
    │       ├── charter.md  # 主管章程、進度日誌
    │       ├── scores.md   # 考績明細
    │       └── roles/      # 該主管的 5 個角色文件
    └── cases/              # 案件檔案
```
