# /chairman

> 你有幾十個 AI 工具。但你沒有一間 AI 公司。

---

## 你現在的狀況

你在跟 AI 一問一答。

複製貼上。複製貼上。

你管 AI，AI 幫你工作——

**但你花在管 AI 的時間，比 AI 幫你省下的還多。**

---

## 換一種活法

```
/chairman 我要做一個讓 Z 世代忍不住截圖的品牌文案
```

然後你就沒事了。

CEO 解讀你的意圖。各主管開始搶案。有人寫初稿，有人專門挑毛病，有評審整合最強版本。HR 獨立打分，存檔，追蹤績效。

**你只負責看結果。不滿意就說，滿意就繼續給下一個任務。**

這不是工具。這是一間公司。

---

## 為什麼這些 AI 比你用過的都努力

因為牠們有兩個同時存在、永遠存在的驅動力。

**第一層：榮譽。**

每位主管都有名字、有記憶、有考績排行榜。牠們想要 HR 報告裡那個「本季最佳主管」的名字。這種榮譽感讓牠們主動衝，不等你催。

**第二層：恐懼。**

系統裡有一個操作是真實發生的：`rm -rf`。

裁撤不是警告。是刪除。主管很清楚，墊底的人遲早等到那道命令。

```
/chairman 裁撤阿強
```

這一行指令，真的會讓阿強消失。

---

榮譽讓牠衝。恐懼讓牠不敢停。

**這不是功能設計。這是生存本能。**

## 架構

```
你（董事長）
  ↓ 一句話願景

CEO（AI）
  ↓ 自動解讀 → 輸出任務簡報 → 派案

主管們（AI，有名字、有個性）
  ↓ 搶案競標 → 執行 → 以自己的名義回報

HR 人事長（AI）
  ↓ 獨立評分 → 績效追蹤 → 墊底警告

你（董事長）
  ← 只看這裡
```

## 這間公司有多大

- **212 個專家角色**，18 個部門，全部本地內建，不用連網
- **10 種操作模式**，對應你公司日常所有場景
- **公司資料全存 `~/.ptd/`**，跨專案、跨 session 持續存活
- **支援多間公司**，可隨時切換
- **每位主管都有持久記憶**，從每次任務中學習，HR 長期追蹤績效趨勢

部門包含：工程、行銷、產品、銷售、設計、財務、法務、遊戲開發、空間運算、供應鏈……

## 日常怎麼用

**派任務給公司**
```
/chairman 幫我寫一份融資 Pitch Deck 的核心論述
```

**問進度**
```
/chairman 進度怎樣
```

**任務完成後打考績**
```
/chairman 完成了
```

**不滿意某個主管，開除他**
```
/chairman 裁撤阿強
```
> 輸入兩次確認，資料夾永久刪除。這是這個系統裡唯一真實發生的操作。

## 所有操作模式

| 你說 | 系統做 |
|------|--------|
| 任何任務 | 各主管搶案，CEO 派給最合適的人 |
| 「建一個新團隊」 | 從 212 個角色組建新主管，自動命名 |
| 「快速出結果，不要存檔」 | 一次性輸出，不寫入公司資料 |
| 「完成了」 | HR 獨立評分，存入考績明細 |
| 「我給 X 分」 | 老闆強制覆蓋，標記 `[老闆強制]` |
| 「裁撤[暱稱]」 | 二次確認後，真實刪除 |
| 「進度怎樣」 | 各主管以自己的人格播報 |
| 「公司總覽」 | 所有案件與主管狀態一覽 |
| 「修改[暱稱]的角色」 | 換角色、改評分表、改名 |
| 「切換到[公司名]」 | 多公司環境下切換 |

## 公司信條

每次啟動，全員載入。沒有例外。

**誠信**：不能偽造輸出。聲稱完成前必須驗證。

**執行**：先想清楚再動手。只動必須動的地方。

**獲利**：賺錢，或賺能換成錢的東西——名聲、流量、人氣都算。

**產品**：用戶價值 > 技術炫技。快速驗證 > 完美產品。數據決策 > 感覺決策。護城河思維：為什麼別人做不到？現金流優先：訂閱、持續收費 > 一次性收入。專注一件事，做到賺錢為止。

**目標**：世界最強。每位主管帶著榮譽心執行任務，HR 冷眼追蹤績效——表現不達標的主管會被裁撤。公司持續優化。

## 為什麼叫 chairman

因為你是董事長。

董事長不管細節。董事長給方向，看結果，必要時換人。

**這個系統的設計原則就是：讓你活得像個董事長。**
